## UX Guidelines (Top 10 for this project)

> Source: ux-guidelines.csv

1. **Loading States** — 분석 중 스피너 + 단계별 진행 텍스트로 기대감 유지
2. **Empty States** — 업로드 전 명확한 CTA 안내 (클릭/드래그 유도)
3. **Error Feedback** — 에러는 red-dim 배경 + 구체적 메시지, 자동 닫힘 없음
4. **Progressive Disclosure** — 항목 상세는 클릭 시 펼침(accordion), 기본 접힘 유지
5. **Color Meaning Consistency** — Green = 좋음, Red = 나쁨을 앱 전체에서 일관 유지
6. **Score Visualization** — 숫자 점수는 항상 색상 뱃지와 함께 표시
7. **Hover States** — 클릭 가능한 모든 요소에 hover 피드백 필수
8. **Animation** — transition 0.15~0.2s ease, transform translateY(-1px) for lift
9. **Accessibility** — 색상만으로 상태를 구분하지 말 것 (텍스트 레이블 병행)
10. **Mobile** — 2열 그리드는 600px 이하에서 1열로 전환

## React Stack Guidelines

> Source: stacks/react.csv

- `useState` — 컴포넌트 로컬 상태 (fileHigh, fileLow, phase 등)
- `useRef` — Chart.js 캔버스, 파일 인풋 ref
- `useEffect` — 차트 초기화/업데이트, 타이머 클린업
- `key` props — 리스트 렌더링 시 `item.id` 사용
- Props drilling — 현재 규모에서는 Context 불필요, props로 유지
- 이벤트 핸들러 — 인라인 arrow function 지양, 함수 선언 후 참조

## Design Consistency Checklist

UI 작업 완료 시 아래 항목 확인:

- [ ] 모든 색상이 CSS 변수(`var(--*)`)로 참조됨
- [ ] 하드코딩된 hex 값 없음
- [ ] 성과 지표: Green(고성과) / Red(저성과) 일관 적용
- [ ] 버튼 hover 상태 구현됨
- [ ] 반응형: 600px 이하 1열 레이아웃 확인
- [ ] 텍스트 크기 계층 준수 (11px~32px)
- [ ] border-radius: 카드 12px, 버튼/인풋 8px, 뱃지 20px
- [ ] 새 컴포넌트에 `border: 1px solid var(--border)` 적용
